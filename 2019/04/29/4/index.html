误删数据了怎么办

发表于 2019-04-29 | | 浏览 次
字数统计: 765 字 | 阅读时长 ≈ 3 分钟

摘要：我劝你在学习更多技能之前，还是先治疗下“手抖”吧。

今天来说一个 DBA 比较讨厌的问题：误删数据了怎么办。你一定会想到传统的恢复方式：全备 + binlog 前滚。那还有没有更快速、简单、灵活的方式呢？今天我就来介绍一个。

MariaDB 从 10.2.4 开始引入了 DML-only flashback 功能，这就是今天的主角。我知道你可能会问：我用的是 MySQL 啊，难道要先迁移到 MariaDB 吗？不不不，DML-only flashback 功能是在工具 mysqlbinlog 里的，只需要把 MariaDB 安装目录下的 /bin/mysqlbinlog 文件复制过来，覆盖掉 MySQL 安装目录下的 /bin/mysqlbinlog 即可。好了，现在执行下 mysqlbinlog --help ，有没有发现多了如下选项：

该选项要如何使用呢？首先得满足两个前提：

  1. binlog_format=ROW，binlog_row_image=FULL；
  2. 目前仅支持DML语句（INSERT，DELETE，UPDATE）。

为什么要满足前提 1 呢？因为只有这样，binlog 里面才会记录被删掉行的整行信息，这是找回数据的关键。DML-only flashback 功能就是利用这些信息，把 delete 转成 insert，把误删的数据插入回去。
